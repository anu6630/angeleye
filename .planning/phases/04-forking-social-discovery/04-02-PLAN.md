# Phase 04-02: Follow System API

## Objective

Implement follow/unfollow API endpoints with rate limiting (100/day per CONTEXT.md D-9), ownership checks, and duplicate prevention. This enables users to build a social graph for personalized feeds.

**Important:** Per CONTEXT.md D-10, follower/following list browsing is NOT in scope for v1. The GET /followers/{id} and GET /following/{id} endpoints return counts only (integers), not full user lists.

Purpose: Provide the backend API for follow functionality that frontend components will consume.

Output: FollowService with business logic, follow API router with POST/DELETE endpoints, count-only GET endpoints for followers/following, proper error handling and validation.

    __init__(self, db: Session):
    - Initialize self.db = db

    follow_user(self, follower_id: int, following_id: int) -> Follow:
    - Check if follower_id == following_id: raise ValueError("Cannot follow yourself")
    - Check if following user exists: get User by following_id, raise ValueError("User not found") if None
    - Check rate limit: query Follow table for COUNT(*) WHERE follower_id=follower_id AND created_at > now() - interval '1 day'
    - If count ≥ 100: raise ValueError("Rate limit exceeded: 100 follows per day")
    - Check for duplicate: query Follow by follower_id and following_id
    - If exists: raise ValueError("Already following this user")
    - Create Follow object: Follow(follower_id=follower_id, following_id=following_id)
    - Add to session, commit, refresh
    - Return Follow object

    unfollow_user(self, follower_id: int, following_id: int) -> bool:
    - Query Follow by follower_id and following_id
    - If not exists: raise ValueError("Not following this user")
    - Delete from session, commit
    - Return True

    get_follow_counts(self, user_id: int) -> dict:
    - Query COUNT(*) from Follow where following_id=user_id (followers count)
    - Query COUNT(*) from Follow where follower_id=user_id (following count)
    - Return {"followers_count": int, "following_count": int}
    - Per CONTEXT.md D-10: Follower lists show count only (not full user lists)

    is_following(self, follower_id: int, following_id: int) -> bool:
    - Query Follow by follower_id and following_id
    - Return True if exists, False otherwise

    Per CONTEXT.md D-9: Follow rate limiting: 100 follows per day per user
    Per CONTEXT.md D-10: Follower lists (show count only) — full user browsing deferred to v2

    POST /api/v1/follows endpoint:
    - Depends: require_auth (returns current_user), get_db (returns db session)
    - Request body: {"following_id": int}
    - Validate: current_user.id != following_id (400: "Cannot follow yourself")
    - Call: FollowService(db).follow_user(current_user.id, following_id)
    - On ValueError (duplicate/rate limit): return 409 Conflict or 429 Too Many Requests
    - On other errors: return 500 Internal Server Error
    - On success: return 201 Created with {"message": "Followed successfully", "following_id": int}

    DELETE /api/v1/follows/{user_id} endpoint:
    - Path parameter: user_id: int (the user to unfollow)
    - Depends: require_auth, get_db
    - Validate: current_user.id != user_id (400: "Cannot unfollow yourself")
    - Call: FollowService(db).unfollow_user(current_user.id, user_id)
    - On ValueError (not following): return 404 Not Found
    - On success: return 200 OK with {"message": "Unfollowed successfully"}

    GET /api/v1/follows/followers/{user_id} endpoint:
    - Path parameter: user_id: int
    - Depends: get_db (no auth required - public profile viewing per AUTH-04)
