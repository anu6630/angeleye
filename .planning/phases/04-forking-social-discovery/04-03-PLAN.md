# Phase 04-03: Forking System API

## Objective

Implement notebook forking API that creates independent copies with full attribution tracking (parent_id, root_id), forks all cells, creates independent dataset copies via S3 server-side copy, and prevents deletion of notebooks with existing forks.

Purpose: Enable the core social behavior of remixing notebooks while preserving attribution lineage.

Output: ForkService with business logic, fork API endpoint, dataset forking via S3 copy, delete protection for notebooks with forks.

    __init__(self, db: Session, storage_service: StorageService):
    - Initialize self.db = db
    - Initialize self.storage_service = storage_service

    fork_notebook(self, notebook_id: int, user_id: int) -> Notebook:
    - Get original notebook by id, raise ValueError if not found
    - Create fork notebook:
      * title = f"{original.title} (fork)"
      * user_id = user_id (current user)
      * parent_id = notebook_id (immediate parent)
      * root_id = original.root_id if original.root_id else notebook_id (ultimate original)
      * is_published = False (forks start as draft)
      * Copy all other fields from original
    - Add fork notebook to session, commit, refresh
    - Copy all cells: Query NotebookCell where notebook_id=notebook_id, iterate and create new cells with notebook_id=fork.id, same order_index, cell_type, content
    - Commit cells
    - If original has dataset (link via notebook_dataset association), call fork_dataset
    - Refresh fork notebook and return

    fork_dataset(self, dataset_id: int, user_id: int) -> Dataset:
    - Get original dataset by id, raise ValueError if not found
    - Generate new S3 key: f"datasets/{user_id}/{user_id}_{timestamp}_{original.original_filename}"
    - Call storage_service.copy_object(original.s3_key, new_s3_key)
    - Create fork dataset:
      * user_id = user_id
      * filename = new_s3_key
      * original_filename = original.original_filename
      * file_size_bytes = original.file_size_bytes
      * content_type = original.content_type
      * s3_key = new_s3_key
      * row_count = original.row_count
      * parent_id = dataset_id (immediate parent)
      * root_id = original.root_id if original.root_id else dataset_id (ultimate original)
    - Add fork dataset to session, commit, refresh
    - Return fork dataset

    get_fork_chain(self, notebook_id: int) -> List[Notebook]:
    - Initialize empty list chain = []
    - Start with current = get_notebook_by_id(notebook_id)
    - While current.parent_id is not None:
      * Add current to chain
      * current = get_notebook_by_id(current.parent_id)
    * Add root (current without parent_id) to chain
    - Reverse chain to show from original to current
    - Return chain

    get_forks(self, notebook_id: int, limit: int = 50) -> List[Notebook]:
    - Query Notebook where parent_id=notebook_id
    - Order by created_at DESC
    - Limit results
    - Return list of fork notebooks

    has_forks(self, notebook_id: int) -> bool:
    - Query COUNT(*) from Notebook where parent_id=notebook_id
    - Return True if count > 0, False otherwise

    can_delete_notebook(self, notebook_id: int) -> bool:
    - Call has_forks(notebook_id)
    - Return True if no forks, False if forks exist

    Per CONTEXT.md D-1: Fork model uses parent_id (immediate) + root_id (ultimate original)
    Per CONTEXT.md D-2: Dataset forking matches notebook model with same structure
    Per CONTEXT.md D-5: Fork chains treated equally - no penalty for depth
