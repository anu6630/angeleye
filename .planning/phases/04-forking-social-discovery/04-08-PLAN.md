    Update frontend/lib/api-client.ts to add new endpoints:

    Fork operations:
    - async forkNotebook(notebookId: number): Promise<NotebookResponse>
      * POST /api/v1/notebooks/{id}/fork
      * Returns forked notebook object
    - async getNotebookForks(notebookId: number, limit: number = 50): Promise<NotebookResponse[]>
      * GET /api/v1/notebooks/{id}/forks?limit={limit}
      * Returns list of fork notebooks
    - async getForkChain(notebookId: number): Promise<NotebookResponse[]>
      * GET /api/v1/notebooks/{id}/chain
      * Returns attribution chain from original to current

    Follow operations:
    - async followUser(userId: number): Promise<{message: string, following_id: number}>
      * POST /api/v1/follows with body {following_id: userId}
      * Returns success message
    - async unfollowUser(userId: number): Promise<{message: string}>
      * DELETE /api/v1/follows/{userId}
      * Returns success message
    - async getUserFollowers(userId: number, limit: number = 50): Promise<UserResponse[]>
      * GET /api/v1/follows/followers/{userId}?limit={limit}
      * Returns list of followers
    - async getUserFollowing(userId: number, limit: number = 50): Promise<UserResponse[]>
      * GET /api/v1/follows/following/{userId}?limit={limit}
      * Returns list of following
    - async checkFollowing(userId: number): Promise<{is_following: boolean}>
      * GET /api/v1/follows/check/{userId}
      * Returns follow status

    Search operations:
    - async searchNotebooks(query: string, tab: string = "all", limit: number = 50): Promise<{notebooks: NotebookResponse[], total: number, empty_state: boolean, message?: string}>
      * GET /api/v1/search?q={query}&tab={tab}&limit={limit}
      * Returns search results with empty state handling

    Engagement metrics:
    - NotebookResponse interface should include: view_count?: number (add if missing)

    Update frontend/stores/social-store.ts to add follow state:

    Add to SocialState interface:
    - followingIds: Set<number> (track who current user follows)
    - followersCount: Record<number, number> (cache follower counts)
    - followingCount: Record<number, number> (cache following counts)

    Add to SocialStore actions:
    - setFollowingIds: (ids: number[]) => void (initialize from API)
    - toggleFollow: (userId: number) => Promise<void> (optimistic update, API call, rollback on error)
    - isFollowing: (userId: number) => boolean (check Set)
    - initializeFollows: () => Promise<void> (load from API on mount)

    Per CONTEXT.md D-6: One-way follow system (Twitter/Instagram style)
    Per CONTEXT.md: Optimistic updates for follow button (immediate UI change)
