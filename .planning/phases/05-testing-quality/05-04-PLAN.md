# Locked Decisions from CONTEXT.md
- D-12: Comprehensive E2E tests with multiple scenarios per flow
- D-13: Chromium-only for E2E (Chrome and Edge are both Chromium)

# E2E Test Patterns
- Use Playwright's auto-waiting (no hard-coded sleeps)
- Test user behavior, not implementation
- Use locators: getByRole, getByText, getByLabelText
- Page object pattern for complex flows (optional, direct interaction is fine)
- Start test server via webServer config
- Use real backend (or mock backend for faster tests)

# 6 Critical Paths
1. Authentication: Login → verify session → logout
2. Notebook creation: Create → add cells → save → compile → publish → view in feed
3. Forking: Fork → verify lineage → edit independently → both in feed
4. Social: Follow → like → comment → verify in feed
5. Search: Search → filter → verify results
6. Feed: Browse → infinite scroll → trending
