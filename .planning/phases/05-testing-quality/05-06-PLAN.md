<task type="auto">
  <name>Task 1: Create main CI workflow with parallel execution</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Create .github/workflows/ci.yml (orchestrates all test types):

    1. Trigger on: push, pull_request to main

    2. Concurrency: Cancel in-progress runs for same branch

    3. Jobs (run in parallel):

       a. backend-tests:
          - Runs: pytest unit + integration tests
          - Uses: matrix strategy for Python versions (3.9 only for v1)
          - Uses: actions/cache for pip dependencies
          - Steps:
            i. Checkout code
            ii. Set up Python 3.9
            iii. Start services: docker-compose up -d postgres redis
            iv. Install dependencies
            v. Run unit tests in parallel: pytest -n auto
            vi. Run integration tests: pytest tests/integration
            vii. Upload coverage to Codecov
          - Timeout: 15 minutes

       b. frontend-tests:
          - Runs: Vitest component + store tests
          - Uses: actions/cache for node_modules
          - Steps:
            i. Checkout code
            ii. Set up Node.js 25
            iii. Install dependencies
            iv. Run tests with coverage: npm run test:coverage
            v. Upload coverage to Codecov
          - Timeout: 10 minutes

       c. e2e-tests:
          - Runs: Playwright E2E tests
          - Steps:
            i. Checkout code
            ii. Set up Node.js 25
            iii. Install dependencies
            iv. Install Playwright browsers: npx playwright install --with-deps chromium
            v. Start backend services (docker-compose)
            vi. Start backend server (background)
            vii. Run E2E tests: npx playwright test
            viii. Upload test results
          - Timeout: 20 minutes

       d. code-quality:
          - Runs: All quality checks in parallel
          - See Task 2 for details

    4. Aggregate coverage job (depends on all test jobs):
       - Downloads coverage artifacts from all jobs
       - Merges coverage reports
       - Checks overall coverage threshold (80% lines, 70% branches)
       - Comments coverage on PR
       - Fails if threshold not met

    5. Status check:
       - Requires all jobs to pass before merge
       - Shows individual job status in PR checklist

    Use workflow_run for triggering from other workflows
  </action>

<task type="auto">
  <name>Task 2: Create comprehensive code quality workflow</name>
  <files>.github/workflows/code-quality.yml</files>
  <action>
    Create .github/workflows/code-quality.yml:

    1. Trigger on: push, pull_request

    2. Jobs (run in parallel):

       a. backend-lint:
          - Runs: Black code formatting check
          - Command: black --check backend/
          - Fail on: Formatting violations

       b. backend-lint-ruff:
          - Runs: Ruff linting
          - Command: ruff check backend/
          - Config: select=E,F,W (errors, flakes, warnings)
          - Fail on: Lint violations

       c. backend-typecheck:
          - Runs: mypy type checking
          - Command: mypy backend/app
          - Config: warn_return_any=true, strict_optional=true
          - Fail on: Type errors

       d. backend-security:
          - Runs: Bandit security scanning
          - Command: bandit -r backend/app -f json
          - Exclude: tests/ directory
          - Fail on: High severity issues
          - Warn on: Medium severity issues

       e. frontend-lint:
          - Runs: ESLint
          - Command: eslint frontend/ --max-warnings 0
          - Config: next/core-web-vitals
          - Fail on: Lint violations or warnings

       f. frontend-typecheck:
          - Runs: TypeScript compiler
          - Command: tsc --noEmit in frontend/
          - Fail on: Type errors

       g. frontend-security:
          - Runs: npm audit
          - Command: npm audit --production
          - Fail on: High/Critical vulnerabilities
          - Warn on: Medium vulnerabilities

       h. dependency-check:
          - Runs: Dependabot check (via GitHub native)
          - Or: npm outdated (check for outdated deps)
          - Warn on: Outdated dependencies

    3. All jobs run independently (no dependencies between them)

    4. Upload lint results as artifacts for PR review

    5. Comment on PR with lint summary (violations count, warnings)

    Use continue-on-error: false for all jobs (strict failure policy)
  </action>
